# 설계핏 AI v2.0

공차를 무조건 타이트하게 잡는 비숙련 설계자의 문제를 줄이기 위한 **공차 최적화 워크벤치**입니다.

V1은 입력폼 기반 리스크 카드에 가까워 실제 현업 가치가 낮았습니다. V2는 "위험합니다"에서 멈추지 않고 **기능을 만족하면서 얼마까지 공차를 완화할 수 있는가**를 제안하는 방향으로 전환합니다.

## V2 핵심 판단

### 생성형 AI만으로는 부족함

생성형 AI는 공차 최적값을 직접 산출하는 엔진으로 쓰기에는 위험합니다. 공차 추천은 계산, 공정 능력, 검사 재현성, 품질 이력, CAE 결과가 필요합니다.

AI는 다음 역할에 적합합니다.

- 계산 결과를 설계자 눈높이로 설명
- 생산기술팀 협의 문구 생성
- 검토 누락 항목 질문
- 리포트 구조화
- OpenAI Structured Outputs 기반 JSON 결과 생성

### CAD/CAE 데이터만으로 시작하면 MVP 비용이 커짐

처음부터 CAD 모델 자동 파싱, 상세 CAE, 사내 품질 DB를 모두 연결하면 MVP 비용과 기간이 커집니다. 초기에는 실제 데이터가 충분하지 않아 정확도도 바로 나오기 어렵습니다.

### V2 권장 방향: 하이브리드

1. 룰 기반 공정 능력 테이블
2. 기능 허용치 입력
3. CAE-lite 간이 계산
4. 추천 공차 범위 산출
5. AI 리포트/협의 문구 보조
6. 사용자가 저장한 검토 케이스를 데이터 자산으로 축적

이 방식이 MVP 개발비를 가장 낮추면서도 제품 방향을 "진짜 쓸 수 있는 도구" 쪽으로 돌립니다.

## 현재 구현된 V2 1단계

- 현재 공차와 기능 허용치 입력
- 재질별 탄성계수/항복강도/가공성 테이블
- 공정별 관리 가능 공차/비용/리드타임 테이블
- 단순보 기반 CAE-lite 변위/응력 근사
- 추천 완화 공차 계산
- 현재 공차 대비 예상 비용 개선 지수
- 공정 후보 비교
- 생산기술 협의 문구 자동 생성
- localStorage 기반 케이스 저장
